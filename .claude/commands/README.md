# 快捷命令目录

本目录包含了AI智能体团队的快捷命令文件，用于更方便地调用各个智能体。

## 快捷命令列表

| 命令 | 智能体 | 描述 |
|------|--------|------|
| `/pm` | 产品经理 | 产品规划、需求分析和路线图制定 |
| `/fe` | 前端开发 | UI实现、组件开发和用户体验优化 |
| `/be` | 后端开发 | API设计、数据库优化和服务器端逻辑开发 |
| `/qa` | QA工程师 | 测试、质量保证和缺陷报告 |
| `/ops` | DevOps工程师 | 部署、基础设施和CI/CD流水线 |
| `/tl` | 技术负责人 | 技术决策和团队协调 |

## 使用方法

所有快捷命令都遵循相同的格式：
```
/[command] "任务描述"
```

例如：
```bash
/pm "设计用户认证系统"
/fe "创建登录表单"
/be "实现JWT API"
/qa "测试认证流程"
/ops "部署到生产环境"
/tl "评估技术架构"
```

## 工作流示例

### 完整功能开发流程
```bash
# 1. 需求分析
/pm "定义用户认证功能需求"

# 2. 后端开发
/be "实现JWT认证API"

# 3. 前端开发
/fe "创建登录UI组件"

# 4. 测试
/qa "执行完整的认证流程测试"

# 5. 部署
/ops "部署认证功能到生产环境"
```

### 并行开发流程
```bash
# 需求定义
/pm "分析实时协作功能需求"

# 并行开发后端和前端
/be "构建WebSocket实时协作API" &
/fe "创建协作编辑界面" &

# 测试集成
/qa "测试实时协作端到端流程"

# 部署
/ops "部署协作功能"
```

## 最佳实践

1. **清晰的任务描述** - 提供具体、可操作的任务
2. **上下文信息** - 包含相关背景和依赖关系
3. **迭代开发** - 对复杂功能分步骤实施
4. **团队协作** - 确保各智能体工作协调一致

## 故障排除

如果快捷命令无法正常工作：

1. 检查文件是否存在于 `.claude/commands/` 目录
2. 确认YAML前置内容格式正确
3. 验证Claude Code是否正确加载命令
4. 可以使用完整的 `/agent [name]` 命令作为替代
