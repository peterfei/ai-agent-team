## 🎯 6大AI角色介绍

### 1. 🎯 产品经理 (`/pm`)
**职责**: 需求分析、竞品研究、产品设计
**使用场景**:
```bash
/pm "设计一个程序员笔记应用的功能需求"
# 输出：用户故事、功能规格、验收标准
```

### 2. 🎨 前端开发 (`/fe`)
**职责**: UI设计、组件开发、性能优化
**使用场景**:
```bash
/fe "创建现代化的登录页面，包含表单验证"
# 输出：React组件代码 + 样式设计 + 交互逻辑
```

### 3. ⚙️ 后端开发 (`/be`)
**职责**: API设计、数据库优化、业务逻辑
**使用场景**:
```bash
/be "实现用户认证API，支持JWT和OAuth2"
# 输出：RESTful API + 数据库设计 + 安全措施
```

### 4. 🧪 测试工程师 (`/qa`)
**职责**: 测试设计、自动化测试、质量保证
**使用场景**:
```bash
/qa "设计用户认证系统的完整测试方案"
# 输出：测试用例 + 自动化脚本 + 性能测试
```

### 5. 🔧 运维工程师 (`/ops`)
**职责**: 部署运维、CI/CD、监控告警
**使用场景**:
```bash
/ops "配置生产环境部署，支持Docker和CI/CD"
# 输出：Dockerfile + CI配置 + 监控方案
```

### 6. 👨‍💼 技术负责人 (`/tl`)
**职责**: 架构设计、技术选型、团队协调
**使用场景**:
```bash
/tl "为SaaS产品设计可扩展的多租户架构"
# 输出：架构设计 + 技术选型 + 实施路线图
```

## 🚀 进阶使用技巧

### 1. 项目完整流程示例

```bash
# 项目：开发一个在线教育平台

# 1. 产品规划
/pm "设计在线教育平台的MVP功能，包括课程管理、用户学习、支付系统"

# 2. 技术架构
/tl "为教育平台设计可扩展的微服务架构，支持高并发访问"

# 3. 并行开发
/be "实现课程管理API，支持视频上传、进度跟踪" &
/fe "创建课程学习界面，支持视频播放和笔记功能" &

# 4. 质量保证
/qa "设计完整的测试方案，包括单元测试、集成测试、端到端测试"

# 5. 部署运维
/ops "配置生产环境，支持CDN加速和自动扩容"
```

### 2. 团队协作模式

```bash
# 让AI角色互相协作，模拟真实团队
/pm "设计用户认证需求"
/tl "基于需求设计技术方案"
/be "实现后端API时参考技术方案"
/fe "前端开发时调用后端API"
/qa "测试前后端集成"
/ops "部署到生产环境"
```

### 3. 技术学习模式

```bash
# 学习新技术栈，比如Go语言
/tl "评估Go语言在Web开发中的优势和适用场景"
/be "使用Gin框架实现RESTful API"
/fe "创建简单的前端界面调用Go API"
/qa "为Go应用编写完整的测试用例"
```
